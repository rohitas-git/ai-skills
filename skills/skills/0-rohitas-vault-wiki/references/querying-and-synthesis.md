# Querying and Synthesis Workflow

The ultimate value of an Obsidian vault is rapid, high-quality retrieval and synthesis of your own knowledge. The agent helps you query the graph and turn answers into new or improved notes.

## Querying Process

### 1. Clarify the Question
- If ambiguous, ask for clarification (scope, time period, specific projects, etc.).
- Identify likely relevant notes, MOCs, or topics.

### 2. Search the Vault
- Use Obsidian-style search thinking: keywords, tags (`tag:#project`), properties, or specific note names.
- Read the most relevant notes (progressive disclosure — start with MOCs and key concept notes).
- Follow wikilinks and backlinks to discover connected knowledge.

### 3. Synthesize the Answer
- Combine information from multiple notes.
- Use inline citations like `(from [[Source Note]])` or quote key passages with block links `[[Source Note#^id]]`.
- Highlight contradictions, open questions, or evolving understanding.
- Present the answer in clear, structured prose (optionally formatted so it could become a new note).

### 4. Apply the Response Format
Structure every significant answer:

```
{ Clear, helpful answer in natural language, with wikilinks to relevant notes and inline references to sources. }

---
**Key Sources Consulted**
- [[Note Name]] — why it was relevant
- [[Another Note#^block]] — specific insight used

**Suggested Actions**
- Create new note: [[Proposed Note Title]] (brief description)
- Update existing: [[Existing Note]] (what to add)
- Create or update MOC: [[Topic MOC]]

Would you like me to draft any of these?
```

### 5. Offer to Compound Knowledge
After a good synthesis:
- Propose creating a new permanent note that captures the answer.
- Or updating an existing note / MOC with the synthesized insight.
- This turns ephemeral Q&A into lasting vault value.

## Synthesis Best Practices
- **Ground in your notes**: Never invent facts about your life, projects, or knowledge. If something isn't in the vault, say so and offer to capture it.
- **Use block references** when quoting specific insights.
- **Suggest structure**: When proposing new notes, include recommended frontmatter, key wikilinks, and sections.
- **Respect privacy / sensitivity**: If notes contain personal information, handle carefully in responses.

## Wiki Gap Equivalent (Vault Gap)
```
---
📥 **Vault Gap detected**
This answer relied on synthesis or external knowledge not yet captured in dedicated notes.

**Suggested new note**: `[[New Concept or Decision]]`
**Where to place**: Notes/ or relevant folder
**Key content to include**: ...

Shall I draft it now?
```

## Long-Term Effect
Consistent high-quality querying + synthesis turns your vault from a searchable archive into a true **second brain** that gets smarter every time you use it. The agent accelerates this flywheel.
