# Skill route surface (butler + lint)

House convention so **`/0-butler`** can route without opening every `SKILL.md`.

**ADR:** [0009-catalog-route-surface-for-butler](../../../docs/adr/0009-catalog-route-surface-for-butler.md)  
**Catalog version:** root [`catalog.yaml`](../../../catalog.yaml) · **Feature log:** [`docs/FEATURE-LOG.md`](../../../docs/FEATURE-LOG.md)

**Decisions (locked):**

1. Structured route data lives **only** under `metadata.catalog`.
2. **No** top-level route keys (`hub`, `when`, `not_when`, `next`, `prev`, `cousins`, `triggers`, `role`) on SKILL frontmatter.

Hub membership SSOT remains `hubs/*/workflow.json` + `flows.md`. Catalog **mirrors** primary hub for routing.

---

## A. Description contract (live `skills/`)

Keep one `description` (folded `>` OK). Include:

```text
[What ≤1 sentence.] Use when: [triggers/symptoms]. Not for: [cousin redirects]. Hub: /0-{hub}.
```

| Signal | Lint |
|--------|------|
| ≥ ~40 chars | `description-quality` (warn) |
| Use when / Use for / Triggers / Trigger on | `description-triggers` (warn; all live skills) |
| High-collision: Not for or body `## Boundary` | `route-cousin` (warn) |

User-invoked skills still keep rich descriptions for **butler and humans** (hosts may strip description from auto-invoke).

---

## B. `metadata.catalog` schema

```yaml
name: 1-tdd
description: >
  … Use when: … Not for: … Hub: /0-implement.
disable-model-invocation: true
metadata:
  catalog:
    hub: 0-implement          # primary parent only (dual membership stays in hubs)
    role: pipeline            # hub | pipeline | on-ramp | leaf | wrapper | soft | satellite | sub-hub | routing | handoff
    when:
      - "short when bullet"
    not_when:
      - "redirect → other-skill"
    next: [1-code-review]     # optional list of skill ids
    prev: [0-implement]
    cousins: [2-verify-work]
    triggers: [tdd, test-first]
    requires_setup: false     # true → hard-dep on /0-setup-rohitas-skills
```

### Field notes

| Field | Required on place (gate-route) | Notes |
|-------|--------------------------------|-------|
| `hub` | yes | Must appear under that hub’s `workflow.json` / flows |
| `role` | yes | Link type / routing role |
| `when` **or** `triggers` | yes (at least one) | Short lists; may echo description |
| `not_when` | recommended for cousins | Prefer with Boundary table |
| `next` / `prev` | pipeline skills | Skill ids with depth prefix |
| `cousins` | collision clusters | |
| `requires_setup` | hard-dep skills | `1-to-spec`, `1-to-tickets`, `0-triage` |

Preserve other `metadata` keys (`author`, `version`, `short-description`, …). Nest `catalog` beside them.

### Forbidden top-level keys

If any of these appear at frontmatter root (not under `metadata.catalog`), lint **`route-top-level` critical**:

`hub`, `when`, `not_when`, `next`, `prev`, `cousins`, `triggers`, `role`, `requires_setup`

## D. Generated route index

```text
skills/0-butler/references/route-index.md
```

- **Do not hand-edit** (header says so).
- Regenerated by `scripts/generate-route-index`.
- Sources (priority): `metadata.catalog` → description parse → hub membership + truncated description.

Butler orient/query: load `flows.md` domain map + this index; open full SKILL.md only after choice.

---

## Authoring checklist (create / place / ingest)

1. Description contract (Use when + Not for + Hub).
2. `metadata.catalog` with `hub`, `role`, `when` or `triggers`.
3. No top-level route keys.
4. Hub slot in `workflow.json` / flows (skill-manager place).
5. Boundary table if cousin collision.
6. Regenerate route index; skill-lint Gate PASS.
